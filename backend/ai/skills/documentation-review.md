# Documentation Review

## Best practices
- Clear README
- Run instructions
- Architecture overview
- Env var list

## Common mistakes
- Stale docs
- Screenshot-only README

## Red flags
- Docs promise features absent from repo

## Evaluation checklist
- [ ] README substance
- [ ] docs/ folder

## Scoring guidance
About operability evidence, not marketing tone.

## Example observation
Install steps mismatch package scripts — weakness.
