# GitHub Repository Review

## Best practices
- Meaningful commit messages
- PR reviews for non-trivial changes
- Branches with clear purpose
- Issues linked to work

## Common mistakes
- Single huge dump commit
- Empty README
- Committed secrets

## Red flags
- Repo created same day as submission with all code
- Copied boilerplate with no substantive change

## Evaluation checklist
- [ ] Commit cadence vs milestone window
- [ ] PR quality
- [ ] Issue hygiene

## Scoring guidance
Health reflects history authenticity and engineering process, not star count.

## Example observation
One commit titled update containing the entire monorepo — GitHub Health low.
